# Feature Envy Frenzy 

The next two weeks are going to be fun! (At least I am planning on having a good time.) We are going to be adding features to your rails apps. What is a feature? I thought you'd never ask! 

At its simplest, a feature is a piece of functionality that an actor can interact with. For example, a user can create a new post, or a party client can make a request to your app's api to get a list of posts. 

You've added the basic functionality to your apps, now it's time to make things a little more interesting. I thought we could go through and brainstorm some ideas. 

Here’s a list of features to whet your appetite. https://gorails.com/series/common-features

## The game:

We are going to play a game I made up called "Feature Envy Frenzy". We're going, [shape each feature](https://basecamp.com/shapeup/0.1-foreword,) build understanding and consensus, build the actual feature, validate the solution and culminating in a pull request.

> The steps are more important than the result. The goal is to practice the process of adding features to an app. I would much prefer that you do one feature well than try to get it done in a rush.


## Feature Pitch

Pick features that are interesting to you. Is there something you've been itching to try? Is there something that you think would be a good fit for the app (your's or the other team's)?  

  - 3 - 5 features for your app
  - 2 features for the other team's app

Do a brief write up of each feature, and how it would work in the context of your application. Submit your feature pitch as a pull request in your app's repository. 

## Feature Appetite

Read and respond to the feature pitches of your fellow interns. What do you think of their ideas? Do you have any suggestions?  Submit your responses in the comments for the issue.


At the end, we are going to vote on which features you are going to add.

## Voting 

Each person get's 3 votes for features for their app, 2 votes for features for other teams' apps. Additionally, each team/app gets 1 veto for a feature that they do NOT want to add to their app.

  - Pick up to 3 features that you would like to add to your app and order them by preference. 
  - Pick up to 2 features to suggest for the other team to add to their app and order them by preference.
  - Pick up to 1 suggested feature to veto from your app's list.

Based on the results, we will decide which features to add to each app.

## Featuring... 

Now we build! Investigate the features on your list and formulate a plan for how you want to implement them. Start a feature branch and get to work! Some things to keep in mind:

- Start with your specification. What is the feature supposed to do? What are the edge cases? What are the acceptance criteria? WRITE IT DOWN. 
  - This will help you later when you are writing tests.
  - This will help you later when you are writing documentation.
  - This will help you later when you are trying to remember what the heck you you were trying to do in the first place.
  
- Don't forget to write tests! When we demo your features, I would like you to use feature tests to demonstrate.

- Some of the GoRails content is a little out of date, and may not be the best way to implement a feature today. Use the GoRails tutorials as a starting point, but make sure you are leveraging the current 'Rails way' to build your features. **[ ** For example: don't use jQuery... leverage Hotwire and Stimulus.js instead.** ]**
